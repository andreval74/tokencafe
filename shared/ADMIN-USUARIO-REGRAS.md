## Regras de Acesso (ADMIN vs USUÁRIO)

Este documento é a referência única para entender o que o ADMIN pode ver/fazer em relação ao USUÁRIO. Sempre que houver mudança de acesso/permissão em qualquer módulo, este arquivo deve ser atualizado junto com a alteração no código.

### Tabela de Regras

| Área/Funcionalidade | USUÁRIO (normal) | ADMIN | Implementação (arquivo/linhas) | Observações |
|---|---|---|---|---|
| **Definição de ADMIN (PHP)** | Não é ADMIN por padrão; depende da wallet. | É ADMIN se a wallet estiver na lista de admins, ou via bypass. | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php) | A lista de admins é composta por um valor fixo + `config/admins.txt` (se existir). |
| **Wallet de ADMIN (JS)** | Não é ADMIN por padrão; depende da wallet. | É ADMIN se a wallet estiver em `ADMIN_WALLETS`. | [admin-security.js](file:///c:/xampp/htdocs/tokencafe/assets/js/shared/admin-security.js#L6-L35) | O front usa uma lista própria (`ADMIN_WALLETS`), independente do PHP. |
| **Cookie de wallet (identidade)** | Cookie `tokencafe_wallet_address` (se existir) define a wallet usada para contexto e validações. | Mesmo. | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php) | Esse cookie é a base para validar admin no PHP e para logs/contexto. |
| **Bypass de ADMIN (cookie)** | Pode virar ADMIN se existir cookie de bypass ativo. | Pode usar bypass. | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php) | Cookie: `tokencafe_admin_bypass` (aceita 1/true/yes/on). |
| **Bypass de ADMIN (admin_key na URL)** | Pode virar ADMIN ao informar `?admin_key=...` válido (grava cookie por 12h). | Pode usar para habilitar bypass na sessão. | [render.php](file:///c:/xampp/htdocs/tokencafe/includes/render.php) | A chave vem de `TOKENCAFE_ADMIN_BYPASS_KEY` (env). |
| **Barreiras de admin (desativação global)** | Não deveria ter acesso de admin. | ADMIN. | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php), [head.php](file:///c:/xampp/htdocs/tokencafe/includes/head.php) | `TOKENCAFE_DISABLE_ADMIN_BARRIERS` deve ficar `false` em produção. Pode ser habilitado por env para testes. |
| **Tools (Home - lista de módulos)** | Vê somente tiles `finished` e que não sejam `adminOnly`. | Vê todos os tiles, inclusive `adminOnly` e `soon`. | [tools.php](file:///c:/xampp/htdocs/tokencafe/modules/site/tools.php) | Para usuário normal, a lista é filtrada no PHP antes de renderizar. |
| **Tools: abrir tile “soon” com link válido** | Não abre (permanece desabilitado). | Se `status=soon` e `href` válido, o tile deixa de ficar desabilitado para ADMIN. | [tools.php](file:///c:/xampp/htdocs/tokencafe/modules/site/tools.php#L19-L27) | Regra específica do render do tile. |
| **Tiles marcados como `adminOnly` (Tools)** | Não aparecem para usuário normal. | Aparecem para ADMIN. | [tools.php](file:///c:/xampp/htdocs/tokencafe/modules/site/tools.php) | Exemplos: Verificação, Logs/Relatórios, Analytics, Widget, Tokens, Templates, Settings, Documentação. |
| **Bloqueio server-side (páginas “admin-only”)** | Se tentar acessar algumas páginas diretamente, é redirecionado para `index.php?page=tools`. | Pode acessar normalmente. | [render.php](file:///c:/xampp/htdocs/tokencafe/includes/render.php) | Páginas bloqueadas para não-admin: `analytics`, `widget`, `templates`, `settings`, `tokens`, `token-add`, `token-manager`, `documentacao`, `verifica`. |
| **Logs: nível “Chief Admin”** | Não é “chief”; não pode investigar outras wallets/IPs. | “Chief admin” pode investigar tudo (filtros por wallet/rede/contrato, top IPs, etc.). | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php), [logs-index.php](file:///c:/xampp/htdocs/tokencafe/modules/logs/logs-index.php) | `Chief` é uma wallet principal (`TOKENCAFE_CHIEF_ADMIN_WALLET`). |
| **Logs: usuário comum com wallet** | Vê relatórios limitados ao próprio contexto (wallet atual). | Se não for chief, também pode ficar limitado; se for chief, vê tudo. | [logs-index.php](file:///c:/xampp/htdocs/tokencafe/modules/logs/logs-index.php) | O filtro de wallet é forçado para usuário comum (não-chief). |
| **Logs: usuário comum sem wallet** | Não consegue ver o relatório (apenas aviso). | ADMIN/Chief pode ver. | [logs-index.php](file:///c:/xampp/htdocs/tokencafe/modules/logs/logs-index.php) | Exige cookie de wallet para usuário normal. |
| **Download de logs (endpoint)** | Sem wallet: 403. Com wallet: só baixa tipos permitidos e sempre filtrado na própria wallet. | Chief: acesso completo aos downloads e filtros. | [download-log.php](file:///c:/xampp/htdocs/tokencafe/download-log.php) | Usuário comum só pode `ip` e `sc`; `wallet` é forçada e `ip` é limpo. |
| **Contrato: “Visualizar Arquivos” do deploy (.sol/.json/ABI/bytecode)** | Oculto (não aparece). | Aparece e permite download/preview. | [contract-actions.php](file:///c:/xampp/htdocs/tokencafe/includes/shared/components/contract-actions.php) | `files-section` recebe `d-none` para não-admin. |
| **Contrato: “Adicionar token” (quick action no contract-search)** | Bloqueado (“apenas administradores”). | Liberado. | [contract-search.js](file:///c:/xampp/htdocs/tokencafe/assets/js/shared/contract-search.js) | Regra aplicada no botão “Add token” (MetaMask). |
| **Contrato: checagem de admin no módulo** | `checkIsAdmin()` retorna `false` (quando barreiras estão ligadas e não-admin). | `checkIsAdmin()` retorna `true` (admin). | [builder.js](file:///c:/xampp/htdocs/tokencafe/assets/js/modules/contrato/builder.js) | A checagem usa `TOKENCAFE_IS_ADMIN`, wallet conectada, cookie e localStorage. |
