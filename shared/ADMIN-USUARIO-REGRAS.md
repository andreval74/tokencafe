## Regras de Acesso (ADMIN vs USUÁRIO)

Este documento é a referência única para entender o que o ADMIN pode ver/fazer em relação ao USUÁRIO. Sempre que houver mudança de acesso/permissão em qualquer módulo, este arquivo deve ser atualizado junto com a alteração no código.

### Tabela de Regras

| Área/Funcionalidade | USUÁRIO (normal) | ADMIN | Implementação (arquivo/linhas) | Observações |
|---|---|---|---|---|
| **Definição de ADMIN (PHP)** | Não é ADMIN por padrão; depende da wallet. | É ADMIN se a wallet estiver na lista de admins, ou via bypass. | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php#L1-L67) | A lista de admins é composta por um valor fixo + `config/admins.txt` (se existir). |
| **Wallet de ADMIN (JS)** | Não é ADMIN por padrão; depende da wallet. | É ADMIN se a wallet estiver em `ADMIN_WALLETS`. | [admin-security.js](file:///c:/xampp/htdocs/tokencafe/assets/js/shared/admin-security.js#L6-L35) | O front usa uma lista própria (`ADMIN_WALLETS`), independente do PHP. |
| **Cookie de wallet (identidade)** | Cookie `tokencafe_wallet_address` (se existir) define a wallet usada para contexto e validações. | Mesmo. | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php#L24-L26) | Esse cookie é a base para validar admin no PHP e para logs/contexto. |
| **Bypass de ADMIN (cookie)** | Pode virar ADMIN se existir cookie de bypass ativo. | Pode usar bypass. | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php#L75-L81) | Cookie: `tokencafe_admin_bypass` (aceita 1/true/yes/on). |
| **Bypass de ADMIN (admin_key na URL)** | Pode virar ADMIN ao informar `?admin_key=...` válido (grava cookie por 12h). | Pode usar para habilitar bypass na sessão. | [render.php](file:///c:/xampp/htdocs/tokencafe/includes/render.php#L452-L466) | A chave vem de `TOKENCAFE_ADMIN_BYPASS_KEY` (env). |
| **Feature Flags (cookie assinado)** | Não deveria conseguir ativar flags (cookie inválido/sem assinatura é ignorado fora de localhost). | Pode ativar flags via painel Admin (Configurações). | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php) | Cookie: `tokencafe_admin_flags` (payload JSON com `exp` + assinatura HMAC quando há chave). |
| **Barreiras de admin DESLIGADAS (PHP)** | Na prática, vira ADMIN. | ADMIN. | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php) | Quando `TOKENCAFE_DISABLE_ADMIN_BARRIERS=true`, `tokencafe_is_admin_wallet()` retorna `true` para todos. Pode vir de env ou flag runtime. |
| **Barreiras de admin DESLIGADAS (JS)** | Na prática, vira ADMIN no front. | ADMIN. | [head.php](file:///c:/xampp/htdocs/tokencafe/includes/head.php), [admin-security.js](file:///c:/xampp/htdocs/tokencafe/assets/js/shared/admin-security.js#L23-L35) | Quando `window.TOKENCAFE_DISABLE_ADMIN_BARRIERS=true`, o front “libera” admin em checagens. Agora é valor dinâmico. |
| **Tools (Home - lista de módulos)** | Vê somente tiles `finished` e que não sejam `adminOnly`. | Vê todos os tiles, inclusive `adminOnly` e `soon`. | [tools.php](file:///c:/xampp/htdocs/tokencafe/modules/site/tools.php#L86-L261) | Para usuário normal, a lista é filtrada no PHP antes de renderizar. |
| **Tools: abrir tile “soon” com link válido** | Não abre (permanece desabilitado). | Se `status=soon` e `href` válido, o tile deixa de ficar desabilitado para ADMIN. | [tools.php](file:///c:/xampp/htdocs/tokencafe/modules/site/tools.php#L19-L27) | Regra específica do render do tile. |
| **Tiles marcados como `adminOnly` (Tools)** | Não aparecem para usuário normal. | Aparecem para ADMIN. | [tools.php](file:///c:/xampp/htdocs/tokencafe/modules/site/tools.php#L117-L239) | Exemplos: Verificação, Logs/Relatórios, Analytics, Widget, Tokens, Templates, Settings, Documentação. |
| **Bloqueio server-side (páginas “admin preview”)** | Se tentar acessar algumas páginas diretamente, é enviado de volta para `tools.php`. | Pode acessar normalmente. | [render.php](file:///c:/xampp/htdocs/tokencafe/includes/render.php#L468-L487) | Páginas bloqueadas para não-admin: `analytics`, `widget`, `templates`, `settings`, `tokens`, `token-add`, `token-manager`, `documentacao`. |
| **Painel Admin (Configurações do Sistema)** | Não acessa (página bloqueada para não-admin). | Acessa e consegue ligar/desligar regras de runtime (ex: barreiras de admin). | [system-settings.php](file:///c:/xampp/htdocs/tokencafe/modules/settings/system-settings.php) | Aba “Admin” salva flags via cookie por 24h. |
| **Logs: nível “Chief Admin”** | Não é “chief”; não pode investigar outras wallets/IPs. | “Chief admin” pode investigar tudo (filtros por wallet/rede/contrato, top IPs, etc.). | [admin-config.php](file:///c:/xampp/htdocs/tokencafe/includes/admin-config.php#L41-L49), [logs-index.php](file:///c:/xampp/htdocs/tokencafe/modules/logs/logs-index.php#L4-L62) | `Chief` é uma wallet principal (`TOKENCAFE_CHIEF_ADMIN_WALLET`). |
| **Logs: usuário comum com wallet** | Vê relatórios limitados ao próprio contexto (wallet atual). | Se não for chief, também pode ficar limitado; se for chief, vê tudo. | [logs-index.php](file:///c:/xampp/htdocs/tokencafe/modules/logs/logs-index.php#L56-L62) | O filtro de wallet é forçado para usuário comum (não-chief). |
| **Logs: usuário comum sem wallet** | Não consegue ver o relatório (apenas aviso). | ADMIN/Chief pode ver. | [logs-index.php](file:///c:/xampp/htdocs/tokencafe/modules/logs/logs-index.php#L516-L521) | Exige cookie de wallet para usuário normal. |
| **Download de logs (endpoint)** | Sem wallet: 403. Com wallet: só baixa tipos permitidos e sempre filtrado na própria wallet. | Chief: acesso completo aos downloads e filtros. | [download-log.php](file:///c:/xampp/htdocs/tokencafe/download-log.php#L10-L43) | Usuário comum só pode `ip` e `sc`; `wallet` é forçada e `ip` é limpo. |
| **Contrato: “Visualizar Arquivos” do deploy (.sol/.json/ABI/bytecode)** | Oculto (não aparece). | Aparece e permite download/preview. | [contract-actions.php](file:///c:/xampp/htdocs/tokencafe/includes/shared/components/contract-actions.php#L5-L15) | `files-section` recebe `d-none` para não-admin. |
| **Contrato: “Adicionar token” (quick action no contract-search)** | Bloqueado (“apenas administradores”). | Liberado. | [contract-search.js](file:///c:/xampp/htdocs/tokencafe/assets/js/shared/contract-search.js#L144-L160) | Regra aplicada no botão “Add token” (MetaMask). |
| **Contrato: checagem de admin no módulo** | `checkIsAdmin()` retorna `false` (quando barreiras estão ligadas e não-admin). | `checkIsAdmin()` retorna `true` (admin). | [builder.js](file:///c:/xampp/htdocs/tokencafe/assets/js/modules/contrato/builder.js#L9-L45) | A checagem usa `TOKENCAFE_IS_ADMIN`, wallet conectada, cookie e localStorage. |
